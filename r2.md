# Walkthrough: SPA Pull Pattern — R2 Sync Engine

## What Was Built

A complete **SPA Pull Pattern** sync engine for Kola-biz. When triggered, it:
1. Reads R2 credentials from `app_settings`
2. Fetches all active products from SQLite
3. For each product — converts & resizes gallery images to WebP (max 1920px) and uploads them to Cloudflare R2 (skipping already-uploaded ones for speed)
4. Generates a flat JSON payload per product and always uploads it fresh (to capture price/spec changes)

Your frontend website can then read `https://<r2-public-url>/products/<uuid>.json` with no live backend.

### `src-tauri/src/commands/products.rs`
Three new items appended at the end:

| Item | Purpose |
|---|---|
| `optimize_image_to_webp(raw)` | Resize to ≤1920px, encode as WebP |
| `build_r2_client(...)` | Creates an S3-compatible client targeting R2 |
| `r2_object_exists(...)` | HEAD check to skip re-uploading existing images |
| `sync_all_to_r2` (Tauri cmd) | Orchestrates the full smart sync loop |

**JSON Schema generated per product:**
```json
{
  "id": "uuid",
  "title": "Manufacturer Model Year",
  "price": 1250000,
  "location": "<company_profile.company_name>",
  "specs": { "manufacturer", "model", "year", "mileage", "fuel", "transmission", "owners", "color" },
  "images": ["https://.../product_images/<slug>/<pid>/<img-id>.webp"]
}
```

## Smart Sync Strategy

| File type | Behaviour |
|---|---|
| Product images | Uploaded once; skipped on re-sync if already in R2 (`HEAD` check) |
| Product JSON | Always re-uploaded to capture price/spec changes |
